# DEQODE EARTH — UX Review
## Audience: Fisheries Officers, MPA Managers, Government Department Operators
## Date: 2026-04-13

---

## Design Verdict

The aesthetic direction is correct and distinctive. Dark navy + teal + Playfair Display SC reads as serious intelligence tool, not generic SaaS. Risk badge system, pulse animations, and mono typography create real credibility. Not generic AI slop — intentionality is present.

**The gap is operational utility, not visual design.**

## UX Gaps — Prioritised Add-ons

### 1 — Collapse Data Specifications table (15 min — HIGH)
Default collapsed behind "View data specifications ▸". Run Analysis becomes the hero action, not something you scroll past 9 rows of technical specs to find.

### 2 — Cache last result in localStorage (1 hour — VERY HIGH)
Most important add-on. Fisheries officer opens the app Monday morning — results from last week should be waiting, not requiring a manual 45-second re-run. Show cached result with "Last run: X days ago — Run again" option. Transforms demo into operational tool.

### 3 — Plain English interpretation rewrites (30 min — HIGH)
Current: "Near-neutral net change (0.0 m). 100% classified as stable."
Better: "No significant coastline change detected. Your shoreline is holding stable between 2019 and 2025."
Usable by all staff regardless of technical background.

### 4 — Progress step indicator during analysis (45 min — MEDIUM)
Replace spinner + "20–60 seconds" with:
`Connecting → Fetching SAR data → Computing changes → Generating map`
Reduces abandonment on slow Pacific internet connections.

### 5 — Replace Pending modules with roadmap teasers (30 min — HIGH)
Current "Pending" (greyed 60%) deflates the platform for fisheries officers who want the Ocean module.
Replace with: "Ocean Intelligence — Automated dark vessel detection across your 390,000 km² EEZ. Launching Q3 2026."

### 6 — Coordinates + share button on results (1 hour — HIGH)
MPA patrol officers need coordinates of detected changes to direct a vessel.
Add: bounding box coordinates on result, share button (copy URL with cached result).

### 7 — Sharpen CRITICAL risk colour (15 min — MEDIUM)
Tuvalu and Kiribati are existential situations. The UI should convey that weight.
Increase coral opacity/brightness for CRITICAL badge specifically.

## Operator Workflow (what the tool should support)

**Monday morning check:**
1. Open app → see cached last result immediately (no manual re-run needed)
2. If stable → nothing to do
3. If change detected → view SAR map → get coordinates → export PDF → send to colleague or patrol vessel

**Post-event (cyclone, suspected IUU activity):**
1. Open app → run fresh analysis
2. See before/after SAR map
3. Export PDF with interpretation for incident report or funding application
